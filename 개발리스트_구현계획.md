# 개발리스트 구현 계획

## 목표

새로 들어온 UI/연출 요구사항을 기존 p5.js 코드 구조 안에서 구현하기 쉽게 정리한다.
이 문서는 목표 설계용이며, 아직 실제 코드는 수정하지 않는다.

## 구현 순서 요약

쉬운 것부터 추천 순서는 다음과 같다.

1. 주인공이 생각할 때 이름 숨김
2. 화자 시각적 구분
3. 도파민 수치 변화량 표시
4. 텍스트 타이핑 효과
5. 로그
6. 배경 전환 효과

## 1. 주인공이 생각할 때 이름 숨김

### 요구사항

주인공의 독백 또는 생각 대사에서는 입력한 이름을 화자명으로 보여주지 않는다.

예:

```text
찬호: (예쁘다.)
```

대신 아래처럼 표시한다.

```text
(예쁘다.)
```

또는 화자칸을 비워둔다.

### 구현 난이도

낮음.

### 구현 방식

현재 화자명은 `game.js`의 `formatSpeaker()`에서 처리하고 있다.
이 함수가 현재는 `speaker`만 받지만, 생각 대사 여부를 판단하려면 `text`도 같이 받아야 한다.

추천 방식:

```text
formatSpeaker(speaker, text)
```

판단 기준:

- `speaker === "주인공"`
- `text.trim().startsWith("(")`
- 또는 `text.trim().startsWith("...")` 같은 독백 규칙을 추가할 수도 있음

이 조건이면 화자명을 빈 문자열로 반환한다.

#### 구현 방식에 대한 수정사항

굳이 함수를 수정하지 않고, 시나리오에서 독백은 발화자를 독백으로 입력하게 하고, 독백 발화자를 받을 시 화자표시를 없앨 것.
즉 시나리오에서도 독백으로 생각되는 부분은 전부 발화자를 수정해야함.


### 영향 범위

- `game.js`
- `TextBox.draw()` 호출부

### 주의점

모든 괄호 대사가 생각 대사인지 확정할 필요가 있다.
지금 시나리오 스타일에서는 괄호 대사가 대부분 속마음이므로 이 기준이 가장 간단하다.

## 2. 화자 시각적 구분

### 요구사항

화자마다 이름표 색을 다르게 해서 누가 말하는지 시각적으로 구분한다.

예:

```text
수진: 분홍 계열
혜지: 보라 계열
건호: 파랑 계열
주인공: 흰색 계열
나레이션: 회색 계열
```

### 구현 난이도

낮음.

### 구현 방식

`ui.js`의 `TextBox.draw(speaker, bodyText)`에 색상 값을 추가로 넘기는 방식이 가장 깔끔하다.

추천 구조:

```text
TextBox.draw(speaker, bodyText, speakerColor)
```

색상 결정은 `game.js`에서 담당한다.

예상 함수:

```text
getSpeakerColor(speaker)
```

#### 구현 방식에 대한 수정사항

각 발화자별로 색상을 확실하게 고정해서 상수로 처리, 굳이 함수를 고치지 않아도 될 것 같음.

색상 맵 예시:

```text
수진: #f48fb1
혜지: #b39ddb
건호: #90caf9
주인공: #ffd166
나레이션: #cfd8dc
기본값: #fff7dc
```

### 영향 범위

- `game.js`
- `ui.js`

### 주의점

이름표 배경색과 글자색의 대비가 충분해야 한다.
현재 대화창이 밝은 흰색 계열이므로, 이름표는 너무 연하면 안 된다.

## 3. 도파민 수치 변화량 표시

### 요구사항

도파민 수치가 변할 때 게이지 오른쪽에 증가량 또는 감소량을 잠시 표시한다.

예:

```text
+5
-10
```

표시는 잠깐 떠 있다가 사라진다.

### 구현 난이도

낮음에서 중간.

### 구현 방식

현재 도파민 수치는 `game.js`의 `addDopamine(amount)`에서 바뀐다.
따라서 이 함수에서 변화량 표시 상태를 저장하면 된다.

추가할 상태 예시:

```text
this.dopamineDeltaPopup = {
  amount: 5,
  startedAt: millis(),
  duration: 900
}
```

`drawStatus()` 또는 `drawMeter()`에서 이 값을 읽어 게이지 오른쪽에 그린다.

표시 방식:

- 증가: 분홍색 또는 노란색 `+5`
- 감소: 파란색 또는 민트색 `-10`
- 시간이 지날수록 위로 살짝 이동
- 시간이 지날수록 투명도 감소

### 영향 범위

- `game.js`

### 주의점

현재 선택지 효과는 `applyEffects()`에서 들어오고, 실제 도파민 변화는 `addDopamine()`으로 모인다.
그러므로 모든 도파민 변화 표시를 놓치지 않으려면 `addDopamine()`을 중심으로 처리하는 게 좋다.

미니게임 내부 도파민 변화는 각 미니게임 클래스에서 따로 관리하므로, 우선 스토리 화면의 도파민 변화부터 적용하는 것이 안전하다.


### 피드백

수치 변화가 잘 보이면 좋을 것 같고, 선택지나 대화에서의 변화만 표현하면 충분함.


## 4. 텍스트 타이핑 효과

### 요구사항

대사가 한 번에 표시되지 않고 글자가 순서대로 타이핑되듯 나타난다.

클릭 동작:

1. 대사가 타이핑 중이면 클릭 시 전체 문장을 즉시 표시한다.
2. 전체 문장이 이미 표시된 상태에서 클릭하면 다음 대사로 넘어간다.

### 구현 난이도

중간.

### 구현 방식

현재 `drawStory()`는 현재 노드의 전체 텍스트를 바로 `TextBox.draw()`에 넘긴다.
여기에 현재 대사의 표시 길이를 관리하는 상태가 필요하다.

추가할 상태 예시:

```text
this.typewriter = {
  nodeKey: "EP1:3",
  visibleChars: 0,
  speed: 1.2
}
```

구현 흐름:

1. 현재 노드가 바뀌면 `visibleChars`를 0으로 초기화한다.
2. 매 프레임마다 `visibleChars`를 증가시킨다.
3. `bodyText.slice(0, visibleChars)`만 대화창에 넘긴다.
4. 클릭 시 타이핑 중이면 전체 표시, 아니면 다음 노드로 이동한다.

### 영향 범위

- `game.js`
- `TextBox.draw()`는 그대로 두고 넘기는 텍스트만 잘라서 전달하면 됨

### 주의점

한글은 자모 단위가 아니라 문자열 인덱스 기준으로 잘리기 때문에 대부분 자연스럽게 보인다.
다만 이모지나 특수문자는 글자 수 계산이 어긋날 수 있지만 현재 시나리오에는 큰 문제가 없다.

선택지 화면에서는 타이핑 효과를 적용하지 않는 것이 좋다.

## 5. 로그

### 요구사항

이전에 지나간 대사를 다시 볼 수 있는 로그 화면을 만든다.

현재 UI 기획안에는 우측 메뉴에 `대사록` 항목이 있으므로, 이 메뉴와 연결하면 자연스럽다.

### 구현 난이도

중간.

### 구현 방식

대사가 화면에 표시될 때마다 로그 배열에 저장한다.

추가할 상태 예시:

```text
this.dialogueLog = [
  {
    speaker: "수진",
    text: "아, 네! 죄송합니다...!",
    episodeId: "EP1",
    nodeIndex: 14
  }
]
```

저장 위치:

- `drawStory()`에서 매 프레임 저장하면 중복이 생긴다.
- 따라서 현재 노드가 처음 표시될 때만 저장해야 한다.
- 텍스트 타이핑 효과의 `nodeKey` 초기화 시점과 같이 처리하면 좋다.

로그 UI:

- `SCENES.LOG` 같은 별도 장면을 추가하거나
- 현재 장면 위에 반투명 패널을 띄우는 방식 중 선택

추천은 반투명 패널 방식이다.

이유:

- 현재 스토리 상태를 유지하기 쉽다.
- 닫으면 바로 원래 화면으로 돌아올 수 있다.
- 별도 씬 전환보다 구현 부담이 적다.

기본 조작:

- 우측 메뉴의 `대사록` 클릭 시 로그 열기
- 마우스 휠 또는 위/아래 키로 스크롤
- 닫기 버튼 또는 ESC로 닫기

### 영향 범위

- `game.js`
- 우측 메뉴 클릭 판정 로직

### 주의점

타이핑 효과보다 먼저 구현해도 되지만, 중복 저장 방지를 생각하면 타이핑 효과의 “현재 노드 키” 관리와 같이 설계하는 편이 좋다.

## 6. 배경 전환 효과

### 요구사항

배경이 바뀔 때 즉시 바뀌지 않고 전환 효과가 들어간다.

필요한 효과:

- 페이드 아웃
- 페이드 인
- 페이드 슬라이드

### 구현 난이도

중간에서 높음.

### 구현 방식

현재 배경은 시나리오의 `background` 노드를 처리할 때 즉시 바뀐다.
즉, `processStoryCommandNodes()` 안에서 `this.backgroundImage`가 바로 교체된다.

전환 효과를 넣으려면 이전 배경과 새 배경을 동시에 들고 있어야 한다.

추가할 상태 예시:

```text
this.backgroundTransition = {
  from: previousImage,
  to: nextImage,
  startedAt: millis(),
  duration: 700,
  type: "fade"
}
```

렌더링 방식:

1. 전환 중이 아니면 기존처럼 현재 배경만 그림
2. 전환 중이면 이전 배경을 먼저 그림
3. 진행률에 따라 새 배경을 투명도 또는 슬라이드 위치를 적용해서 그림
4. 시간이 끝나면 전환 상태를 비우고 새 배경만 유지

효과 타입:

```text
fade: 이전 배경 위에 새 배경을 alpha 0 → 255로 표시
fadeOutIn: 화면을 검게 덮었다가 새 배경 표시
fadeSlide: 새 배경이 오른쪽 또는 왼쪽에서 살짝 밀려오며 alpha 증가
```

### 영향 범위

- `game.js`
- `BackgroundImage.draw()` 또는 새 배경 렌더 헬퍼
- 시나리오 `background` 노드 처리부

### 주의점

`processStoryCommandNodes()`는 대사 노드가 나올 때까지 여러 명령 노드를 한 번에 처리한다.
배경 전환을 비동기로 보여주려면, 전환 중에는 다음 대사로 바로 넘어가지 않게 잠시 멈출지 결정해야 한다.

쉬운 방식:

- 배경 이미지는 즉시 교체하되, 화면 렌더링에서만 이전 배경과 새 배경을 겹쳐 보여준다.
- 스토리 진행은 막지 않는다.

더 완성도 높은 방식:

- 배경 전환이 끝날 때까지 대사 표시를 잠시 대기한다.
- 대신 구현 복잡도가 올라간다.

### 피드백

모든 배경 전환 사이에 매우 짧게 검은 배경을 넣어서 만드는 것은 어떠한지?
배경 전환이 끝날 때까지 대사 표시를 잠시 대기하는 건 필수적일 듯

## 추천 구현 단계

### 1차 작업

빠르게 체감되는 UI 개선 위주.

1. 주인공 생각 화자명 숨김
2. 화자별 이름 색상
3. 도파민 변화량 표시

### 2차 작업

대화 연출 개선.

1. 텍스트 타이핑 효과
2. 클릭 시 타이핑 스킵/다음 대사 이동 분기
3. 로그 저장 구조 추가

### 3차 작업

조금 더 큰 UI 기능.

1. 로그 패널 UI
2. 로그 스크롤
3. 우측 메뉴의 `대사록`과 연결

### 4차 작업

화면 전환 연출.

1. 기본 페이드 인/아웃
2. 배경 전환 시 이전/새 이미지 동시 렌더링
3. 페이드 슬라이드 타입 추가

## 최종 우선순위 판단

가장 먼저 구현하면 좋은 것은 `주인공 생각 화자명 숨김`이다.
이유는 코드 변경량이 가장 작고, 시나리오 표현이 바로 자연스러워진다.

그 다음은 `화자 시각적 구분`이다.
이미 `TextBox.draw()`가 있으므로 이름표 색만 확장하면 된다.

세 번째는 `도파민 수치 변화량 표시`다.
현재 도파민 변경 함수가 비교적 명확하게 모여 있어서 붙이기 쉽다.

타이핑 효과와 로그는 서로 연결해서 설계하는 편이 좋다.
타이핑 효과에서 현재 대사 노드를 추적하게 되면, 로그 중복 저장 문제도 같이 해결할 수 있다.

배경 전환 효과는 가장 마지막에 하는 것을 추천한다.
시각 효과 자체보다, 시나리오 명령 노드 처리 흐름과 충돌하지 않게 만드는 부분이 더 중요하기 때문이다.
