# 도파민때문에 작업 인수인계

작성일: 2026-05-28

## 목적

루트의 `도파민때문에_시나리오.xlsx`를 게임에서 읽을 수 있는 시나리오 데이터로 변환하고, 기존 게임 코드가 엑셀 기반 시나리오를 자연스럽게 재생하도록 맞췄다.

## 주요 산출물

- `../도파민때문에_시나리오_follow.xlsx`
  - 원본 엑셀을 복제한 파일.
  - 직접 판별한 follow 행을 EP2 방식처럼 `-`로 표시했다.
- `story-data-excel.js`
  - 엑셀 시나리오를 게임용 JS 데이터로 변환한 파일.
  - `STORY_START_EPISODE`, `EPISODES`를 정의한다.
- `story-validator.js`
  - 시나리오 데이터 검증 함수를 `game.js`에서 분리한 파일.
  - 게임 구동 필수 로직이 아니라 데이터 점검용이다.

## 수정된 코드 흐름

- `index.html`
  - `story-data-excel.js`와 `story-validator.js`를 로드한다.
- `game.js`
  - 시작 에피소드는 `STORY_START_EPISODE`가 있으면 그 값을 사용한다.
  - 대사 및 follow 행의 `effects`를 진행 중 적용한다.
  - 플레이어 이름 치환은 `000`, `OO`를 대상으로 한다.
  - 캐릭터 배치 계산에서 실제로 그릴 수 있는 캐릭터만 가운데 정렬 대상으로 삼는다.
  - 도파민 준비 씬의 장식 도형을 제거했다.
  - 도파민 수치를 바 형태 UI로 표시한다.
  - save 버튼을 우측 버튼 묶음 안으로 정리했다.
  - `그만하기` 버튼 문구를 `건너뛰기(개발단계용)`으로 바꿨다.
- `config.js`
  - 엑셀에 있는 배경명을 현재 가진 임시 배경 에셋에 매핑했다.
  - 향후 실제 배경, 사운드, 감정명은 엑셀 명칭 그대로 추가할 수 있게 두는 방향이다.

## 데이터 변환 규칙

- 변수명과 선택지/조건의 기준 이름은 엑셀 쪽 표기를 따른다.
- follow 행은 복제 엑셀에서 `-` 표시를 기준으로 데이터화했다.
- 일반 대사 행에서도 상태 변화가 적용될 수 있게 지원한다.
- 엑셀의 `00` 계열 플레이어 이름 표기는 게임에서 플레이어 이름으로 치환한다.
- `주인공`의 `character in` 등장은 제거했다.
  - 주인공 스프라이트가 없어서 화면 중앙 정렬과 등장 처리에 방해가 되기 때문.

## 최근 확인한 선택지 이슈

`EP3 동아리 MT`의 `id: 32` 선택지가 테스트 중 뜨지 않는 문제가 있었다.

원인:
- 선택지 조건이 `dopamineMax: 50` 또는 `dopamineMin: 80`만 있어서 도파민 51~79 구간에서 표시 가능한 선택지가 0개였다.

수정:
- `EP3 동아리 MT #32`의 높은 도파민 선택지 조건을 `dopamineMin: 51`로 바꿨다.
- 같은 패턴의 빈 구간이 있던 아래 선택지도 함께 보정했다.
  - `EP4-A 넷이 #23`
  - `EP4-B 둘이 #26`

검증 결과:
- 전체 선택지 노드에서 도파민 0~100 어느 값에서도 표시 선택지가 0개가 되는 경우가 없음을 확인했다.
- `story-data-excel.js`, `game.js`, `story-validator.js` 문법 검사를 통과했다.
- 시나리오 검증 경고와 오류는 0개였다.

## 다음 작업 후보

- 임시 배경 매핑을 실제 배경 에셋으로 교체한다.
- 엑셀에 있는 사운드명, 감정명에 맞춰 실제 에셋과 캐릭터 표정을 추가한다.
- `story-data-excel.js`가 수동 수정된 상태이므로, 엑셀에서 다시 생성할 때 현재 보정 사항이 덮어써지지 않도록 변환 스크립트 쪽에도 규칙을 반영하는 것이 좋다.
- 브라우저에서 전체 에피소드 흐름을 플레이 테스트하며 선택지 이동, dopamine/affection 변화, 엔딩 분기를 확인한다.
