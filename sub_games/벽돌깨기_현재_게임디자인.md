# 도파민 벽돌깨기 현재 게임 디자인

작성 기준 파일: `sub_games/breakblocks.js`

이 문서는 현재 코드에 구현된 도파민 벽돌깨기의 실제 동작을 정리한 것이다. 앞으로 수정할 때는 이 문서를 기준으로 변경할 규칙을 지정한다.

## 1. 게임 역할

도파민 벽돌깨기는 본게임 중간에 실행되는 미니게임이다.

본게임에서 현재 도파민 값을 받아 시작하고, 미니게임 종료 시 최종 도파민 값을 본게임에 돌려준다.

```js
new BrickBreakerGame(initialDopamine)
```

종료 후 본게임은 아래 메서드로 도파민 값을 가져간다.

```js
getDopamine()
```

미니게임 내부에서는 도파민이 `0~120`까지 움직일 수 있다. 다만 본게임으로 돌아갈 때는 `game.js`에서 `0~100`으로 제한된다.

- 요청: 게임에서도 0 ~ 100으로 해야해.

## 2. 기본 화면 크기와 보드

게임 내부 기준 크기는 다음과 같다.

| 항목 | 값 |
| --- | --- |
| 게임 영역 너비 | `980` |
| 게임 영역 높이 | `640` |
| 컬럼 수 | `9` |
| 컬럼 간격 | `76` |
| 블럭 시작 x | `138` |
| 블럭 시작 y | `110` |
| 바닥선 y | `560` |

실제 캔버스 중앙에 게임 영역을 맞추기 위해 매 프레임 `offsetX`, `offsetY`를 계산한다.

## 3. 초기 상태

`resetGame()`에서 초기화되는 주요 값은 다음과 같다.

| 상태 | 초기값 | 의미 |
| --- | --- | --- |
| `dopamine` | 본게임에서 받은 `initialDopamine` | 현재 도파민 |
| `score` | `0` | 미니게임 점수 |
| `turn` | `1` | 현재 턴 |
| `balls` | `[]` | 발사된 공 목록 |
| `bricks` | `[]` | 현재 보드의 블럭 목록 |
| `items` | `[]` | 현재는 사용하지 않음 |
| `aiming` | `true` | 조준 가능 여부 |
| `gameOver` | `false` | 게임 종료 여부 |
| `settleTimer` | `0` | 턴 종료 대기 카운터 |
| `overloadTurns` | `0` | 도파민 과부하 지속 턴 수 |

초기화 직후 블럭 3줄을 생성한다.

```js
for (let r = 0; r < 3; r++) this.addBrickRow(r);
```
- 요청: 초기화시 턴수를 정할 수 있게 해줘

## 4. 턴 구조

현재 게임은 턴제 벽돌깨기다.

1. 플레이어가 마우스로 발사 방향을 정한다.
2. 클릭하면 현재 도파민에 따라 공 여러 개가 순차 발사된다.
3. 모든 공이 바닥 아래로 사라질 때까지 기다린다.
4. 짧은 정산 시간 후 턴이 1 증가한다.
5. 기존 블럭이 한 줄 아래로 내려간다.
6. 새 블럭 줄이 맨 위에 생성된다.
7. 다시 조준 상태가 된다.

- 요청: 가장 첫 공이 바닥에 닿을 때 그곳이 기준점이 되어서 다음 턴을 거기서 시작할 수 있게 할 것

턴은 최대 10턴까지 진행된다. `turn > 10`이 되면 생존 성공으로 종료된다.

## 5. 조작

### 마우스

조준 중일 때 마우스 위치를 향해 조준선을 표시한다.

클릭하면 공을 발사한다.

발사 기준점은 화면 하단 중앙 근처다.

```js
const origin = createVector(this.w / 2, this.floor - 18);
```

발사 방향이 너무 아래쪽이면 발사하지 않는다.

```js
if (dir.y > -20) return;
```

### 키보드

게임오버 상태에서 `R` 또는 `Enter`를 누르면 `finished = true`가 된다.

본게임 또는 테스트 하네스는 이 값을 보고 미니게임을 종료 처리한다.

## 6. 공 발사 규칙

발사되는 공 개수는 현재 도파민에 따라 결정된다.

```js
const count = constrain(floor(map(this.dopamine, 0, 100, 1, 9)), 1, 9);
```

- 요청: 공 개수의 간격을 도파민 5씩 설정해줘

즉 도파민이 낮으면 공이 적고, 도파민이 높으면 공이 많다.

| 도파민 경향 | 공 개수 경향 |
| --- | --- |
| 낮음 | 적음 |
| 높음 | 많음 |

각 공은 같은 방향으로 발사되지만 `ballLaunchGap = 8` 프레임 간격으로 순차 활성화된다.

공 속도는 현재 고정값이다.

```js
vx: dir.x * 8.6
vy: dir.y * 8.6
```

## 7. 공 물리

현재 공 물리는 단순 반사 방식이다.

| 충돌 대상 | 처리 |
| --- | --- |
| 좌우 벽 | `vx *= -1` |
| 상단 벽 | `vy *= -1` |
| 바닥 아래 | 공 제거 |
| 블럭 | `vy *= -1`, 블럭 타격 처리 |

블럭 충돌은 원형 충돌이 아니라 블럭 사각 영역 안에 공 중심점이 들어갔는지 검사한다.


## 8. 블럭 생성 규칙

블럭은 `addBrickRow(rowIndex)`에서 생성된다.

각 컬럼마다 66% 확률로 블럭이 생긴다.

```js
if (random() > 0.66) continue;
```

생성되는 블럭 종류는 두 가지다.

| 종류 | 내부값 | 생성 확률 | 색상 | 효과 |
| --- | --- | --- | --- | --- |
| 자극 블럭 | `stim` | 72% | 분홍 `#f56f83` | 맞을 때마다 도파민 증가 |
| 회복 블럭 | `recover` | 28% | 민트 `#63c7a4` | 파괴될 때 도파민 감소 |

### 자극 블럭

자극 블럭은 체력이 `1~3` 사이로 생성된다.

도파민 증가량은 `+1`, `+2`, `+3` 중 하나다.

공에 맞을 때마다 도파민이 오른다.

```js
this.dopamine = constrain(this.dopamine + brick.value * this.turnBuff.boost, 0, 120);
```

### 회복 블럭

회복 블럭은 체력이 `2~4` 사이로 생성된다.

도파민 감소량은 대략 `-10`, `-15`, `-20`, `-25` 중 하나다.

공에 맞을 때마다 바로 도파민이 줄지는 않는다. 체력이 0 이하가 되어 파괴될 때만 도파민이 줄어든다.

```js
if (brick.kind === "recover" && brick.hp <= 0) {
  this.dopamine = constrain(this.dopamine + brick.value, 0, 120);
}
```

- 요청: 생성확률, 도파민 증가 감소량, 내구도들을 계산해서 평균적으로 50~80의 도파민을 얻을 수 있도록 조절할 것


## 9. 블럭 표시 방식

블럭에는 아래 형식으로 텍스트가 표시된다.

```text
효과값 / 남은 체력
```

예시:

```text
+2 / 3
-15 / 2
```

## 10. 턴 버프

현재 아이템을 직접 먹는 구조는 없다.

대신 공을 발사할 때마다 이번 턴 버프가 랜덤으로 하나 정해진다.

```js
this.turnBuff = this.randomBuff();
```

| 버프명 | 확률 | 내부 효과 |
| --- | --- | --- |
| 완화 큐브 | 22% | 이번 턴 자극 블럭 도파민 증가 없음 |
| 고자극 큐브 | 21% | 자극 블럭 도파민 증가량 `1.7배` |
| 강화 큐브 | 21% | 블럭 체력 감소량 `2` |
| 없음 | 36% | 기본값 |

현재 `items` 배열은 선언만 되어 있고 실제 필드 아이템으로 사용하지 않는다.

- 요청: 블럭들 사이에 아이템들이 생성되게 해주고, 시각적으로 바로 아이템의 효과가 적용된 것을 알아볼 수 있게 해줘. 공의 색을 바꾼다던지, 블럭의 색을 바꾼다던지. 아이템 종류도 더 고려해봐.


## 11. 점수 규칙

점수는 미니게임 내부 표시용이다. 본게임에는 전달되지 않는다.

| 행동 | 점수 |
| --- | --- |
| 블럭 타격 | `+10` |
| 회복 블럭 파괴 | 추가 `+20` |

자극 블럭 파괴 보너스는 따로 없다.

- 요청: 점수제는 제거

## 12. 도파민 규칙

도파민은 미니게임의 핵심 수치다.

| 구간 | 미터 색상 | 의미 |
| --- | --- | --- |
| `0~30` | 파랑 `#68a7ff` | 낮은 도파민 |
| `31~70` | 초록 `#55c99f` | 안정 구간 |
| `71~100` | 주황 `#ffb64d` | 높은 도파민 |
| `101~120` | 빨강 `#ff5b6c` | 과부하 |

미니게임 내부 도파민은 `0~120`으로 제한된다.

도파민이 `100`을 넘은 상태로 턴 종료가 발생하면 `overloadTurns`가 1 증가한다. 도파민이 `100` 이하로 내려가면 `overloadTurns`는 0으로 초기화된다.

## 13. 종료 조건

현재 종료 조건은 세 가지다.

| 조건 | 종료 메시지 |
| --- | --- |
| 10턴을 넘김 | `10턴 생존 완료` |
| 도파민 `100` 초과 상태가 턴 종료 기준 2번 지속 | `도파민 과부하가 오래 지속됨` |
| 블럭이 바닥선 근처까지 내려옴 | `블럭이 바닥에 닿음` |

종료되면 `gameOver = true`가 된다.

게임오버 화면에서 클릭, `R`, `Enter` 입력을 하면 `finished = true`가 된다.

- 요청: 블럭이 바닥선으로 내려올 때, 결과가 영 좋지 않았으면 좋겠어. 플레이어가 게임을 일부러 끝내지 않게.

## 14. 본게임 복귀 방식

미니게임 클래스 자체는 본게임을 직접 알지 못한다.

미니게임은 종료 준비가 되면 `finished = true`만 설정한다.

본게임의 `game.js`가 이 값을 감지해서 `finishSubGame()`을 실행하고, `getDopamine()`으로 최종 도파민을 가져간다.

## 15. 현재 구현의 특징

- 실제 벽돌깨기라기보다 `턴제 발사 퍼즐`에 가깝다.
- 도파민이 높을수록 공이 많아져 더 많은 블럭을 칠 수 있다.
- 도파민이 높으면 강해지지만 과부하 위험도 커진다.
- 도파민을 낮추려면 회복 블럭을 완전히 파괴해야 한다.
- 아이템 시스템은 기획상 이름만 남아 있고, 실제 필드 아이템으로 구현되어 있지 않다.
- 턴 버프는 플레이어가 선택하거나 획득하는 것이 아니라 발사 시 랜덤으로 정해진다.
- 점수는 표시만 되고 결과 판정이나 본게임에는 영향을 주지 않는다.

## 16. 수정 논의가 필요한 지점

아래 항목은 다음 수정 요청에서 결정하면 좋다.

- 공 개수를 도파민에 비례시킬지, 별도 성장 요소로 둘지
- 회복 블럭을 파괴 시점이 아니라 타격 시점에 발동할지
- 필드 아이템을 실제로 생성하고 먹는 구조로 바꿀지
- 10턴 생존을 승리 조건으로 유지할지
- 도파민 안정 구간 유지 보너스를 넣을지
- 점수를 본게임에 반영할지, 도파민만 반영할지
- 도파민 과부하 종료 조건을 완화하거나 강화할지
- 블럭 생성 확률과 자극/회복 비율을 조정할지
- 마스코트나 UI 안내를 추가할지
